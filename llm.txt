# evtlog

Production-ready logging and error event layer for TypeScript backends, built on Pino.

## Commands
bun test — Run tests
bun run typecheck — tsc --noEmit
bun run lint — ESLint
bun run check — lint + format:check + typecheck + test
bun run build — Build to dist/

## Key API
createEvtlog({ service, level, transport, redact, mixin, client, ... }) -> Evtlog
evtlog.info("event.name", { data }) — Event-name-first logging
evtlog.child({ requestId }) — Child logger with bound context
evtlog.scope("module") — Module-scoped child logger (adds module field)
safeError(err) — Strip stack/ internals from Error for safe API responses

## Sub-modules (deep imports)
evtlog/audit — auditLogger(evtlog) with AuditEvent structure
evtlog/security — securityLogger(evtlog) with SecurityEvent and severity levels
evtlog/webhook — Batch log delivery to webhook endpoints with HMAC SHA-256 signing and retry
evtlog/otel — OpenTelemetry bridge: injects trace_id/span_id, optional span events
evtlog/sampling — Deterministic or custom log sampling to control volume
evtlog/adapters/hono — requestIdMiddleware + httpLoggerMiddleware
evtlog/adapters/express — requestIdMiddleware + httpLoggerMiddleware
evtlog/adapters/fastify — requestIdHook + httpLoggerHook

## Redaction
Built-in denylist: password, secret, apiKey, token, ssn, email, phone, cardNumber, etc. (40+ fields)
Recursive, case-insensitive. Add extra fields via redact option or use redactPaths for path-level control.

## Patterns
Event-name-first API wraps Pino internally.
Child loggers inherit parent options (redaction, transport, mixin).
Transport: single, array, or { targets: [...] } format. Supports pino-roll for rolling files.
Cloud ingestion with client: buffers 100 events, flushes on beforeExit.
